# v0.3.0 WorkOrder 质量治理、安全风险分级与文档级 rerank

## 版本主题

本版本在 v0.2.0 已完成的 RAG 安全硬约束、工业术语保留、统一校验与二次补答闭环基础上，继续做三类收敛：工单推荐不再过度敏感，Safety Guard 不再把普通资料题都套入高风险模板，检索来源增加文档意图识别与 rerank，降低相似手册之间的混淆。

本版本没有继续扩大 P0 repair 能力，也没有把具体评测题答案硬编码进业务逻辑。新增判断主要基于问题类型、风险意图、结构化 payload 完整度和文档实体规则。

## 已完成能力

- WorkOrder 推荐质量治理：
  - 区分 `should_create_workorder`、`recommend_workorder`、`ready_to_create`。
  - 新增 `work_order_quality_check`，只有 payload 足够完整时才允许直接创建工单。
  - `payload_preview` 补充 `inspection_steps`、`key_parameters`、`safety_actions`、`source_chunk_ids`、`missing_fields`。
  - 普通参数题和手册概览题默认不直接 `ready_to_create`，除非存在明确故障、拆卸、更换、无法运行、冒烟、失效、危险等执行意图。

- Safety Guard 风险意图分级：
  - 新增 `risk_level` 和 `risk_reasons`。
  - Level 0：普通资料问答，不触发硬安全模板。
  - Level 1：维护、检查、参数操作相关但无立即执行风险，允许轻量 `safety_notes`。
  - Level 2：高风险操作、故障处置、禁止判断，强制输出 `operation_allowed` 和 `safety_actions`。
  - 保留 `must_have_safety` 兼容逻辑，但不再让它直接等价于 Level 2。

- DocumentIntent 与文档级 rerank：
  - 新增 `DocumentIntentService`，根据问题识别品牌、型号、设备类型和优先文档。
  - 对 FANUC、bfp、AH3、CM2、CG1、摩托车发动机等文档做 score boost / rerank。
  - 不过滤非目标文档，避免资料不足时丢失可补充来源。
  - 在 debug 中输出 `document_intent`、`matched_entities`、`preferred_documents`、`rerank_applied`、`rerank_reason`。

- 回归修复：
  - 修复 v0.3.0 第一阶段中 `smoke-003` 因工业术语原词缺失导致的 answer 回退。
  - 新增 `_ensure_matched_terms_preserved()`，在 repair 后仍缺失工业术语原词时追加简短术语保留说明，不放宽评分规则。

## 已知边界

- 本版本不实现 WorkOrder update，仍保留 create/get/list。
- 本版本不继续扩大 P0 repair，仅修复术语保留导致的回退。
- DocumentIntent 使用规则化实体识别与 score boost，不强制过滤其他来源。
- WorkOrder recommendation 是建议和 payload 预览，只有显式 `auto_create_workorder=True` 且 `ready_to_create=True` 时才应落库。

## 版本结论

v0.3.0 的重点是把 v0.2.0 的“能生成、能校验、能补答”继续收敛为“推荐更克制、风险分级更准确、检索来源更稳定”。当前真实工业评测保持 24/24 通过，同时降低了普通参数题、手册概览题被误推荐为工单或误套高风险安全模板的概率。

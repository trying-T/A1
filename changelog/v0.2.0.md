# v0.2.0 RAG 安全增强、术语保留与工单建议闭环

## 版本主题

本版本在 v0.1.0 的 RAG 检修问答最小闭环基础上，补齐了工业安全硬约束、工业术语保留、答案校验与二次补答、以及 WorkOrder 创建建议链路。重点不是增加某些固定题目的答案，而是把 RAG 输出变成可校验、可修复、可审计、可沉淀为工单的数据结构。

## 已完成能力

- 新增 Safety Guard 硬约束：识别安全题，补齐 `operation_allowed`、`immediate_actions`、`prohibited_actions`、`required_personnel`、`risk_keywords`、`manual_basis`、`safety_actions` 等结构化安全字段。
- 新增工业术语保留机制：通过 `backend/app/resources/industrial_terms.json` 配置术语 canonical 原词、中文解释、aliases 和原词保留要求。
- RAG prompt 会在生成前注入本轮命中的工业术语，生成后会校验英文原词是否保留。
- 新增 `AnswerRepair` 二次补答机制：当关键词、安全硬约束或工业术语保留不通过时，使用同一批检索资料进行一次修复。
- 新增统一答案校验：RAG 主流程汇总 `keyword_check`、`safety_check`、`term_check`、`workorder_check`，并在 `validation` 与 `debug` 中记录 repair 前后结果。
- 新增 `work_order_recommendation`：RAG 响应会给出是否建议创建工单、推荐原因和 `payload_preview`。
- 新增可选 `auto_create_workorder`：仅当请求显式传入 `auto_create_workorder=true` 且 recommendation ready 时，才自动创建 WorkOrder。
- WorkOrder 增加 `safety_actions` 保存链路，并兼容旧 SQLite 表自动补列。

## 重要技术改进

- 修复 `EmbeddingError` fallback 分支缺少 `matched_industrial_terms` 导致潜在 500 的问题；embedding 失败时仍返回完整响应结构和 validation/debug。
- 将原先基于固定高风险关键词的 `SafetyService` 收敛为轻量去重，安全规则改由 `SafetyGuard` 统一处理。
- `TermPreserver` 接入工业术语配置 alias，避免依赖评测集 `expected_keywords` 才能识别关键术语。
- `scripts/eval_real_industry_dataset.py` 与 `scripts/eval_rag_workorder.py` 已适配新增 validation、安全字段和 `safety_actions`。
- 新增单元测试覆盖安全硬约束、工业术语保留、RAG 主流程 fallback、repair、工单建议与自动创建。

## API 与数据结构变化

- `RepairChatRequest` 追加字段：
  - `required_keywords`
  - `question_type`
  - `must_have_safety`
  - `should_create_workorder`
  - `auto_create_workorder`
- `RepairChatResponse` 保留原字段，并追加：
  - 安全结构字段：`operation_allowed`、`immediate_actions`、`prohibited_actions`、`required_personnel`、`risk_keywords`、`manual_basis`、`safety_actions`
  - `work_order_recommendation`
  - `work_order`
  - `validation`
  - `debug`
- `WorkOrderCreateRequest` / `WorkOrderItem` 追加 `safety_actions`。

## 已知边界

- WorkOrder 暂不支持 update，本版本只保留 create/get/list。
- `auto_create_workorder` 是显式开关，默认不落库，仍保留原有 `/workorders/create` 独立接口。
- `work_order_recommendation.payload_preview` 是推荐预览，不等价于已创建工单。
- 工业术语表是配置化机制的起点，后续可以继续扩展更多行业术语。

## 版本结论

v0.2.0 将 RAG 问答从“能生成结构化检修建议”推进到“能校验、能修复、能保留关键工业术语、能输出安全硬约束、能沉淀工单建议”的阶段。当前实现重点保持通用规则和配置化术语，不把具体评测题答案硬编码进业务逻辑。

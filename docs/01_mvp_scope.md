# MVP 范围说明

## 1. MVP 目标

在初赛阶段完成一个可稳定演示的设备检修知识检索与作业辅助系统最小闭环，确保评委能够清晰看到“资料入库、知识问答、作业指引、记录沉淀”这条主链路。

## 2. 本阶段必须完成

### 2.1 知识资料管理

- 支持上传检修手册、故障案例、作业规程、安全规范等资料。
- 支持查看已上传文档及其处理状态。

### 2.2 文档解析与入库

- 支持对 PDF、Markdown、TXT 等常见文档做解析。
- 支持对文档内容进行切分并保存为知识片段。
- 支持将知识片段写入向量库。

### 2.3 检修问答

- 用户输入故障现象或检修问题。
- 系统检索相关知识片段。
- 系统生成结构化回答，至少包含：
  - 问题结论或总体建议
  - 检修步骤
  - 安全注意事项
  - 依据来源

### 2.4 作业记录

- 支持将一次问答结果生成检修记录。
- 支持查看历史检修记录。

### 2.5 演示能力

- 准备 3 到 5 个稳定的演示问题。
- 具备基础可视化页面，支持答辩演示。

## 3. 本阶段暂不实现

- 复杂权限管理和多角色审批流。
- 知识图谱的自动构建与图谱推理。
- 复杂多 Agent 协同。
- 视频理解与长时序现场作业分析。
- 自训练行业模型。
- 高复杂度分布式部署。

## 4. 成功标准

- 用户可以完成资料上传与基本知识入库。
- 用户可以基于知识库获得相对稳定的检修建议。
- 回答结果带有来源与安全提示。
- 系统可以生成检修记录并用于演示。
- 代码结构支持后续增加多模态输入和更复杂的推理能力。
